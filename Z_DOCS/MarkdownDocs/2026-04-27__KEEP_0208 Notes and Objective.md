Since we're going "General Purpose" with the sproc, there are three specific "gotchas" to keep in mind when running PowerShell from xp_cmdshell:
1. The "Double Quote" Escape Dance
xp_cmdshell passes a string to the CMD shell, which then passes it to PowerShell. If your @SQLCommand has single quotes (e.g., WHERE County = 'King'), PowerShell handles them fine, but if you need double quotes inside that command, things can break.
* The Fix: Stick to single quotes for your SQL strings. If you must use double quotes in PowerShell, use the "backtick-quote" (`") to escape it from the PS interpreter.
2. Date Format "Drift"
When Invoke-Sqlcmd hands a DateTime object to Export-Csv, PowerShell often formats it based on the local system's regional settings (e.g., MM/DD/YYYY vs DD/MM/YYYY). This can cause "date-shifting" errors in my analysis.
* The Fix: In your "General Purpose" SQL select, always CONVERT(VARCHAR(10), DateColumn, 120) to force the ISO YYYY-MM-DD format. It's the only way to be 100% sure the CSV remains "mathematically defensible" across different environments.
3. The PowerShell "Memory Hog"
Invoke-Sqlcmd is a "load-to-memory" cmdlet. If you try to extract 5 million rows in one go (like the entire State dataset), PowerShell might hit a memory cap and crash without telling the SQL agent.
* The Fix: For the really big files (like Group 99), stick to your current strategy of "Grouping." If a file exceeds ~500MB, it's safer to split it.

    @ScriptName NVARCHAR(100),
    @OutputFileName NVARCHAR(255)

CREATE PROCEDURE dbo.ExecuteScriptFile
    @ScriptFileName NVARCHAR(255),
    @OutputFileName NVARCHAR(255)
AS
BEGIN
    SET NOCOUNT ON;

    DECLARE @ScriptPath NVARCHAR(500) = 'D:\AIProject\Scripts\' + @ScriptFileName;
    DECLARE @OutputPath NVARCHAR(500) = 'D:\Temp\AIExchange\GEMAI_' + @OutputFileName + '.csv';
    DECLARE @PSCommand NVARCHAR(MAX);

    -- PowerShell reads the SQL file content and then runs it
    SET @PSCommand = 'powershell.exe -ExecutionPolicy Bypass -Command "' +
        '$sql = Get-Content -Path ''' + @ScriptPath + ''' -Raw; ' +
        '$data = Invoke-Sqlcmd -ServerInstance ' + @@SERVERNAME + ' -Database ' + DB_NAME() + ' -Query $sql; ' +
        '$data | Export-Csv -Path ''' + @OutputPath + ''' -NoTypeInformation -Encoding utf8;"';

    EXEC xp_cmdshell @PSCommand;

    PRINT 'Executed ' + @ScriptFileName + ' and saved results to ' + @OutputPath;
END
